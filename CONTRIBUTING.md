# Contributing to localWiki Community

Thank you for helping improve localWiki. Here's how to make your contribution count.

## Reporting bugs

A good bug report includes:

1. **localWiki version** — check Settings > About
2. **Your platform** — Windows, macOS (Intel or Apple Silicon), Linux distro
3. **Steps to reproduce** — the clearer, the faster we can fix it
4. **Error reference code** — if you see an `LW-xxxxxx` code, include it
5. **Screenshots** — if the issue is visual

## Requesting features

Before requesting a feature:

- Search existing issues to see if someone already suggested it
- Explain the **problem** you're solving, not just the solution you want
- Tell us which part of localWiki this affects (pages, keys, CLI, sync, etc.)

## Documentation feedback

Found a typo, a confusing section, or missing info in [the docs](https://localwiki.app/docs)?

- Use the "Documentation Feedback" template
- Include the page URL
- If you have a suggested fix, include the corrected text

## What we won't accept

- Issues unrelated to localWiki
- Support requests that are answered in the documentation
- Spam, self-promotion, or low-effort posts
- Issues created by automated tools or bots

## Response expectations

We're a small team. We read everything but may take a few days to respond. Detailed, well-structured issues get prioritized.
